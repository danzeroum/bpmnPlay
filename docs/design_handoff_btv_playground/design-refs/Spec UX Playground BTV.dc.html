7e Regras de fronteira — o contrato playground ⇄ biblioteca
1 · SÓ API PÚBLICA
Imports da raiz dos pacotes (ou /viewer). Guard de CI vetando caminhos internos. O que a apiSurface congela é o que existe.
2 · VERSÃO PINADA
Git tag hoje; semver npm pós-I-6. Versão visível no footer. Upgrade = PR dedicada com changelog lido — nunca «latest».
3 · DEMONSTRA, NÃO ESTENDE
Zero lógica de negócio própria; extensão só via contratos (plugins, adapters, resolvers, AIProvider). Cola de host é bem-vinda; fork de internals é vetado.
4 · GAP = ISSUE UPSTREAM
Faltou API? Issue no danzeroum/bpmn com permalink de repro do próprio playground. Workaround interno é dívida proibida — o playground é o detector de gaps da lib.
Ordem vinculante: P-0 reconciliação (estado real do bpmnPlay) → P-1 shell+consumo+guard → P-2 cenários de modelagem/simulação (modelar · viagem · alçada · simular) → P-3 governança (ciclo + ledger + interop) → P-4 IA (copiloto BYO-key + agente→humano) → P-5 permalink+deploy+polish Cercas: 4 regras de fronteira · ferramenta REAL no centro (nunca réplica) · fake provider default · chave só em memória · reset isolado · roteiro = e2e
